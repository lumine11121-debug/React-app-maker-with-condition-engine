# FULFILLER 

### 1. Name, version, complexity & purpose
- **Name**: first_install_bulk_sync
- **Version**: v1
- **Complexity**: complex
- **Purpose**: Download entire dataset from remote database to local IndexedDB on first launch, with progress and resumability.

### 2. Trigger conditions (ALL must be satisfied; use OR for alternatives)
- No local cache exists (`firstSyncComplete` is false): Yes
- Remote database supports paginated queries (orderBy + startAfter): Yes

### 3. When NOT to choose (disqualifiers)
- Dataset size exceeds 10k records and pagination not implemented in v1 → disqualify (risk of timeout/read budget exhaustion).
- User expects immediate use without any loading screen → disqualify.

### 4. Dependencies (all required; OR allowed)
- indexed_db_setup (stores: `orders`, `syncMeta`)
- (firebase_initialisation) OR (API with cursor pagination)

### 5. Edge cases / known limits (including graceful failure)
- If daily read budget exhausted, sync incomplete → retries next day, shows partial data warning.
- Interruption (browser close) → next open resumes from last saved batch (if batching implemented).
- Graceful failure: loading screen stays; user sees “Sync incomplete, will retry”.

### 6. Verification steps (user test)
1. Clear IndexedDB.
2. Open owner app → loading screen appears.
3. After completion, dashboard loads with all historical orders.
4. Check `syncMeta.firstSyncComplete` is true and `lastSyncTimestamp` is set.

### 7. Implementation sketch
Required. Batching (simplified):
```js
let lastDoc = null;
do {
  let q = query(ordersRef, orderBy("timestamp"), startAfter(lastDoc), limit(500));
  const snapshot = await getDocs(q);
  const batchOrders = snapshot.docs.map(...);
  await writeBatch(batchOrders); // atomic transaction per batch
  lastDoc = snapshot.docs[snapshot.docs.length - 1];
} while (!snapshot.empty);
