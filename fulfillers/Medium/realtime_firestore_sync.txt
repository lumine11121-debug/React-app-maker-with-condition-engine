# FULFILLER 

### 1. Name, version, complexity & purpose
- **Name**: realtime_firestore_sync
- **Version**: v1
- **Complexity**: medium
- **Purpose**: Subscribe to live updates for a time‑bounded query (e.g., today’s orders) and update UI automatically.

### 2. Trigger conditions (ALL must be satisfied; use OR for alternatives)
- User needs live updates for a specific time window (e.g., current day): Yes
- Remote database supports real‑time listeners (Firestore): Yes

### 3. When NOT to choose (disqualifiers)
- Offline mode required for this view (listener requires network) → disqualify.
- User only needs static, once‑fetched data → disqualify.

### 4. Dependencies (all required; OR allowed)
- firebase_initialisation
- Composite index on (timestamp, shopId) in Firestore

### 5. Edge cases / known limits (including graceful failure)
- Listener consumes reads: initial snapshot + each change.
- Network drop → listener errors; UI shows error banner; reconnects automatically.
- Graceful failure: error displayed, but cached data (if any) remains.

### 6. Verification steps (user test)
1. Open owner dashboard, select “Day” period.
2. In another window, add an order as cashier.
3. Dashboard updates within seconds.
4. Disable network → error banner appears; re‑enable → updates resume.

### 7. Implementation sketch
Optional (medium, non‑obvious part is cleanup). Pattern:
```js
useEffect(() => {
  const q = query(collection(db, "orders"), where("timestamp", ">=", todayStart));
  const unsubscribe = onSnapshot(q, (snap) => {
    const orders = snap.docs.map(...);
    setData(deriveAllSections(orders));
  });
  return () => unsubscribe();
}, []);
