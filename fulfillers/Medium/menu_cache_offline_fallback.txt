# FULFILLER 

### 1. Name, version, complexity & purpose
- **Name**: menu_cache_offline_fallback
- **Version**: v1
- **Complexity**: medium
- **Purpose**: Show cached menu immediately when offline, then update from network when available.

### 2. Trigger conditions (ALL must be satisfied; use OR for alternatives)
- Cashier screen displays menu: Yes
- Offline availability is required: Yes
- Menu data is small (<100 items): Yes

### 3. When NOT to choose (disqualifiers)
- Menu frequently changes while offline and users require real‑time updates → disqualify.
- Menu size >1000 items (full cache may be heavy) → disqualify.

### 4. Dependencies (all required; OR allowed)
- indexed_db_setup (menuCache store)
- firebase_initialisation (for onSnapshot)

### 5. Edge cases / known limits (including graceful failure)
- First launch offline → no cache → empty menu (owner must add items online first).
- Cache may be stale if owner updates menu while cashier offline; updates appear when online.
- Graceful failure: if IndexedDB fails, fallback to network only; if network fails, empty menu.

### 6. Verification steps (user test)
1. Load cashier screen online → menu appears and cache populated.
2. Go offline, refresh cashier → menu appears from cache.
3. While offline, toggle an item’s out of stock in owner dashboard.
4. Go online on cashier → menu updates to reflect change.

### 7. Implementation sketch
Optional (medium). Core:
```js
useEffect(() => {
  loadCache();
  const unsubscribe = onSnapshot(menuQuery, async (snap) => {
    const items = snap.docs.map(...);
    setMenuItems(items);
    await updateCache(items);
  });
  return () => unsubscribe();
}, []);
